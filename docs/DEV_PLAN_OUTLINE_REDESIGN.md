# 大纲-细纲-章节 全面重构计划

> 创建时间：2026-05-22
> 最后更新：2026-05-25
> 背景：当前大纲/细纲/章节三个模块存在数据断链、UI布局混乱、AI生成入口定位不清等问题，需要整体重构。

---

## 一、现状问题

### 1. 解析器灾难（阻塞级）
- AI 生成卷级大纲后，解析器把 AI 的开场白（"好的，这是一份基于'1'这个主题..."）当成卷标题
- "世界观设定："、"故事核心："、"卷级大纲："、"章节大纲："等 AI 输出的小标题全被当成卷节点创建
- 章节标题带 markdown 格式（`**宁静的溪谷与不速之客**`），没有清洗
- `**章：边境流亡者**` 这种残缺格式也混入节点

### 2. 大纲页面
- "AI生成卷级大纲"输出内容混乱，解析后产生大量垃圾节点
- 每卷 ✨ 按钮生成的是"章节大纲"，但生成面板跑到页面最顶部，不清楚属于哪个卷
- 整体布局上下堆叠，没有侧栏+编辑区结构
- 文本框固定高度，无法自适应内容

### 3. 细纲页面
- 所有卷的章节节点平铺成一个列表，第1卷的第2章和第2卷的第2章都叫"第2章"，无法区分归属
- 细纲的层级定位不清楚，缺少"故事块"这一中间层

### 4. 章节与正文断链
- 大纲里的章节节点（`outlineNodes` type=chapter）与章节列表页（`chapters` 表）是两个独立数据表，互不感知
- 大纲里建了章节节点，章节列表页永远显示"还没有章节"
- 从大纲点"写作"按钮直接跳正文编辑器，跳过了章节管理层
- 章节页和正文页是两个独立标签，来回跳转体验差

---

## 二、目标信息架构

```
大纲（侧栏+编辑区）
├── 左侧栏：卷列表（可拖拽排序）
└── 右侧编辑区（选中卷的详情）
    ├── 卷标题 + 摘要
    ├── 故事块列表（起承转合/三幕/自定义，Phase 后期）
    └── 章节节点列表

章节（侧栏+编辑区，合并原"正文"标签）
├── 左侧栏：按卷分组的章节列表
└── 右侧编辑区（选中章节的详情）
    ├── 章节标题 + 摘要（来自大纲节点，可编辑）
    ├── 场景列表（AI拆场景 / 手动添加，原细纲功能移入此处）
    └── 正文编辑器（直接嵌入，不再独立标签）
```

## 三、分阶段开发计划

---

### Phase 0 — 解析器修复 + 垃圾数据清理（阻塞级，最先做）

**目标：** 修复 AI 输出解析器，防止继续生成垃圾节点，清洗已有问题数据。

**具体工作：**

1. **重写 `parseVolumeOutlineOutput`**
   - 过滤 AI 开场白（"好的"、"以下是"、"根据你的" 等套话行）
   - 只匹配真正的卷标题（"第X卷"、"卷X"），忽略"世界观设定"、"故事核心"等非卷标题
   - 标题去除 markdown 格式（`**`、`##`、`【】` 等）
   - 摘要部分也要去除 markdown 格式标记

2. **重写 `parseChapterOutlineOutput`**
   - 同样过滤开场白
   - 标题去除 markdown 格式
   - 过滤掉 AI 总结性语句（"以上是..."、"希望这..."）

3. **采纳前预览确认**
   - 采纳按钮点击后，先弹出预览面板展示"将要创建的节点列表"
   - 用户确认后才真正写入数据库
   - 防止垃圾数据再次产生

**涉及文件：**
- `src/lib/ai/parse-outline-output.ts` — 重写两个解析函数
- `src/components/outline/OutlinePanel.tsx` — 采纳预览逻辑

---

### Phase 1 — 通用面板布局组件（基础设施，先于 UI 重构）

**目标：** 提供可复用的侧栏+编辑区布局组件，后续大纲、章节面板直接使用。

**具体工作：**

1. **通用布局组件 `PanelLayout`**
   - Props: `sidebar`, `content`, `defaultSidebarWidth`, `minSidebarWidth`, `maxSidebarWidth`
   - 侧栏宽度可通过拖拽分割线调整
   - 侧栏可折叠/展开
   - 响应式：窄屏时侧栏自动收起

2. **Auto-resize textarea 组件**
   - 高度随内容自适应
   - 可设最小/最大高度
   - 替换所有固定 `rows` 的 textarea

**涉及文件：**
- `src/components/shared/PanelLayout.tsx` — 新建
- `src/components/shared/AutoResizeTextarea.tsx` — 新建

---

### Phase 2 — 大纲页面重构（侧栏布局 + AI 生成修复）

**目标：** 左侧卷列表，右侧选中卷的详情编辑区，AI 生成就地显示。

**具体工作：**

1. **UI 改为侧栏+编辑区**
   - 使用 Phase 1 的 `PanelLayout`
   - 左栏：卷列表（可拖拽排序、可添加/删除）
   - 右侧：选中卷的标题、摘要（用 AutoResizeTextarea），下方是该卷的章节列表

2. **AI 生成入口重组**
   - 两种生成模式让用户选择：
     - "AI 生成全部卷大纲"（生成所有卷的标题+摘要）
     - "AI 生成本卷章节"（在选中卷的详情区内触发，生成该卷的章节列表）
   - AI 生成面板在右侧就地显示，不再跑到页面顶部
   - 采纳流程：AI 输出 → 解析预览 → 确认写入

3. **章节列表展示优化**
   - 每个章节标题前加卷内序号（如"第1章"、"第2章"）
   - 章节可拖拽排序
   - 每个章节行可编辑标题和摘要

**涉及文件：**
- `src/components/outline/OutlinePanel.tsx` — 重构为侧栏布局
- `src/lib/ai/adapters/outline-adapter.ts` — AI 生成模式调整

---

### Phase 3 — 章节+正文合并 + 数据联动

**目标：** 章节页直接嵌入正文编辑器和场景拆分功能，废除独立"正文"标签和独立"细纲"标签。

**具体工作：**

1. **章节列表改为侧栏+编辑区布局**
   - 使用 `PanelLayout`
   - 左侧栏：按卷分组展示所有章节（从 outlineNodes 读取，按 volume → chapter 层级分组）
   - 右侧编辑区：三层结构——
     - 顶部：章节标题 + 摘要（从 outlineNode 读取，可编辑回写）
     - 中部：场景列表（原细纲的"AI一键拆场景"+"添加场景"功能移至此处，可折叠）
     - 底部：正文编辑器（原 ChapterEditor 直接嵌入）

2. **大纲节点与章节记录联动**
   - `chapters` 表新增 `outlineNodeId` 字段
   - 章节列表页读取 `outlineNodes`（type=chapter）作为左侧栏列表
   - 点击章节 → 通过 `outlineNodeId` 查找对应 `chapter` 记录
   - 如果不存在则自动创建（空内容的 chapter 记录）
   - 大纲里的"写作"按钮改为跳转到章节页并选中对应章节

3. **场景功能移入章节页**
   - `detailedOutlines` 表不动，数据结构不变
   - AI拆场景、手动添加场景 UI 从 `DetailedOutlinePanel` 提取为独立 `ScenePanel` 组件
   - 嵌入章节详情区的中间区域

4. **导航调整**
   - 左侧导航合并/移除"细纲"和"正文"标签
   - 可保留"正文"作为"全屏写作模式"的入口（可选）

**涉及文件：**
- `src/components/chapters/ChapterListPanel.tsx` — 大改，侧栏布局+编辑区
- `src/components/editor/ChapterEditor.tsx` — 作为子组件嵌入
- `src/components/outline/ScenePanel.tsx` — 新建，从 DetailedOutlinePanel 提取场景功能
- `src/lib/db/schema.ts` — chapters 表新增 outlineNodeId 字段
- `src/stores/chapter.ts` — 新增按 outlineNodeId 查找/创建的方法
- `src/stores/outline.ts` — 可能需要新增按卷分组查询
- `src/App.tsx` 或路由文件 — 导航调整

---

### Phase 4 — 故事块系统（锦上添花）

**目标：** 在每个卷下面可选添加"故事块"（起承转合/三幕/自定义），章节挂在故事块下。

**具体工作：**

1. **数据模型**
   - 在 `outlineNodes` 表新增 type=`storyBlock`
   - 层级结构：卷(volume) → 故事块(storyBlock) → 章节(chapter)
   - 故事块可选，不加故事块时章节直接挂在卷下（向前兼容）

2. **UI**
   - 大纲卷详情里，可添加故事块（选择结构模板）
   - 内置模板：三幕式（铺垫/对抗/解决）、起承转合、序破急、自定义
   - 故事块下列出章节节点，可拖拽排序
   - 章节页左侧栏也体现故事块层级

3. **AI 生成**
   - 新增 prompt：输入卷摘要 → 输出故事块划分+每块的章节建议
   - 新增 prompt 模板 `outline.storyBlock`

**涉及文件：**
- `src/lib/types/index.ts` — OutlineNode.type 新增 `storyBlock`
- `src/components/outline/OutlinePanel.tsx` — 故事块渲染
- `src/lib/ai/adapters/outline-adapter.ts` — 故事块生成 prompt
- `src/lib/ai/prompt-seeds.ts` — 新增 outline.storyBlock 模板

## 四、执行顺序

| 顺序 | 阶段 | 核心价值 | 预估工作量 |
|------|------|---------|-----------|
| 1 | **Phase 0：解析器修复** | 阻塞级 bug，不修则每次 AI 生成都产生垃圾 | 小 |
| 2 | **Phase 1：通用布局组件** | 基础设施，后续面板都依赖 | 小 |
| 3 | **Phase 2：大纲 UI 重构** | 流程源头，大纲可用后才能谈后续 | 中 |
| 4 | **Phase 3：章节+正文合并** | 解决数据断链+写作体验 | 大 |
| 5 | **Phase 4：故事块系统** | 高级功能，锦上添花 | 中 |

## 五、关键决策记录

- **细纲页面**：场景功能移入章节详情页，细纲标签移除（或降级为可选总览）
- **正文页面**：编辑器嵌入章节页，不再独立存在。可选保留"全屏写作模式"
- **outlineNode 与 chapter 的关系**：一对一，通过 `outlineNodeId` 外键关联。outlineNode 是"蓝图"（标题+摘要），chapter 是"内容"（正文）
- **章节列表数据源**：左侧栏从 `outlineNodes`（type=chapter）读取，右侧编辑器从 `chapters` 表读取/创建
- **故事块数据模型**：使用 outlineNode type=storyBlock，复用现有层级，不新建表
- **侧栏宽度可调**：从 Phase 1 的 PanelLayout 就内置，不延后
- **采纳前预览**：所有 AI 生成的采纳操作都先预览再确认，防止垃圾数据
