## 待开发 — Phase 25: 地理系统重构 + 重要地点

> 来源：社区反馈 + 产品梳理（2026-05-27）
> 优先级：高（现有结构有冗余和 bug）

### 问题

1. **自然环境 vs 地理环境重叠**：自然环境已有世界结构/大陆分布/山川河流/气候等，地理环境面板功能定位模糊
2. **两套地图功能冲突**：地理环境的「AI概念地图」与世界地图的 Voronoi 引擎重叠，用户不知道用哪个
3. **世界地图双主世界 bug**：`ensureRootWorld` 存在竞态条件，React StrictMode 下产生两个根世界
4. **多世界/诸天流方案未设计**：世界树只是简单的树结构，不读世界观数据，无法支撑诸天流等多世界观设定

### 计划

- **25.1** 修复世界地图双主世界 bug（`ensureRootWorld` 加锁/幂等）
- **25.2** 删除「地理环境」面板，地理总述合并到自然环境
- **25.3** 创作区新增「重要地点」模块
  - 地点类型 = 自然地形标签 + 人文场所标签（多标签组合，非下拉）
  - 自然地形：大陆、半岛、岛屿、群岛、高原、平原、盆地、丘陵、峡谷、山脉、山峰、火山、戈壁、沙漠、冰原、草原、森林、雨林、沼泽、绿洲、洞穴、海洋、海峡、海湾、湖泊、河流、瀑布、温泉、冰川、浮空岛、虚空、异界裂隙等
  - 人文场所：村庄、城镇、城市、都城、部落、营地、关隘、要塞、军营、战场、神殿、寺庙、学院、集市、酒楼、拍卖行、黑市、矿场、港口、驿站、废墟、遗迹、古墓、迷宫、禁地、秘境、宗门、洞府、灵脉等
  - 标签可组合（如「海岛 + 集市」=海岛上的集市；「海洋 + 海岛」=海洋中的海岛）
  - 支持树状父子层级（海洋 → 海岛 → 集市）
  - 三种视图：树状图、列表、（未来可接入世界地图标注）
- **25.4** 多世界方案设计文档（仅设计，不写代码；想清楚诸天流的数据结构再动手）

## 待开发 — Phase 26: 灵感反推（逆向创作流程）

> 来源：社区反馈（2026-05-27）—— 用户希望先有角色/灵感再反推世界观
> 优先级：中

### 问题

当前创作流程是自上而下的：世界观 → 故事设计 → 角色 → 大纲 → 章节。但很多作者的创作起点是一个灵感、一个角色、一个情节甚至一个结局，然后再搭世界观。目前系统不支持这种逆向流程。

### 计划

- **26.1** 「灵感起点」入口（项目概况或独立面板）
  - 用户写几句话描述想法（如"退伍军人穿越修仙世界"、"女法医性格冷淡"）
  - AI 根据碎片信息反向生成：世界观草稿、故事核心、初始角色卡
  - 生成结果填入对应模块，用户可二次编辑
- **26.2** 大纲生成注入角色上下文
  - 当前大纲生成只读世界观和故事核心，**不读角色数据**
  - 改造：`buildVolumeOutlinePrompt` 和 `buildChapterOutlinePrompt` 增加 `characters` 参数
  - 如果用户已创建角色，大纲生成时自动注入角色摘要

## 待开发 — Phase 27: AI Agent 化（历史考证 + NPC 自动演进）

> 来源：社区反馈（2026-05-27）—— 用户希望 AI 在构思时主动联想历史背景
> 优先级：低（需要 tool calling / agent 架构重构）

### 问题

1. 当前 AI 调用是单轮流式聊天，无法多步推理或调用外部工具
2. 历史题材用户希望：构思场景时 AI 主动搜索并联想符合历史背景的点子（需要搜索整合 + 上下文抽取，token 消耗大）
3. NPC 自动演进：主角遇到的 NPC，AI 在后台自动推演其故事线，在未来某章突然跳出来提供灵感（需要后台异步 agent）

### 计划（远期）

- **27.1** 架构评估：当前流式聊天 → 支持 tool calling 的 agent 模式，评估重构成本
- **27.2** 历史考证助手：构思场景时 AI 自动检索世界观中的历史设定 + 联想相关史实
- **27.3** NPC 自动演进：后台异步推演 NPC 故事线，在合适时机向作者推荐
